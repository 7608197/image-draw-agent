## 项目简介
这是一个毕业设计项目：**图生文 + 文生图闭环系统**。
技术栈：**FastAPI + React + TypeScript + Ant Design**。

核心能力：
1. `/reverse`：上传图片 -> 返回 caption + 结构化提示词 + 编译后 prompt
2. `/generate`：上传 JSON（可含 structured）+ 可选参数 -> 生成图片并返回 image_url 与生效参数

---

## 目录结构（核心）
bishe/
- backend/
  - main.py                  # API 入口、参数归一化、风格预设、路由
  - schemas.py               # Pydantic 数据结构定义
  - services/
    - reverse_service.py     # 图生文主逻辑（stub/gemini/cliproxy）
    - banana_service_flow2.py# 文生图主逻辑（sd/gemini/proxy）
    - image_response_parser.py # 解析多种响应格式中的图片URL/base64
- frontend/
  - src/
    - App.tsx                # 主界面（图生文页 + 文生图页）
    - services/api.ts        # 前端 API 封装
    - types/reverse.ts       # TS 类型定义
    - utils/prompt.ts        # structured prompt 编译/复制
- tests/
  - test_reverse.py
  - test_generate.py

---

## 后端关键流程
### 1) 图生文 `/reverse`
- 文件：`backend/main.py`
- 路径：`/reverse`
- 校验图片大小与类型 -> 调 `reverse_service.reverse_image` -> 返回 `ReverseResponse`

### 2) 文生图 `/generate`
- 文件：`backend/main.py`
- 路径：`/generate`
- 参数优先级：**表单参数 > structured.params > 默认值**
- structured 会先编译 prompt，再调用 `banana_service.generate(...)`
- 返回 `meta.effective_params / reproducibility / prompt_source`

---

## 模式说明
`banana_service_flow2.py` 支持三种 mode：
- `sd`：本地 Stable Diffusion（可控性最高，seed 可复现）
- `gemini`：云端生成
- `proxy`：远程兼容接口生成

---

## 运行方式
- 后端：`python backend/main.py`（当前端口 8001）
- 前端：`cd frontend && npm install && npm run dev`
- 前端通过 `VITE_API_BASE_URL` 指向后端

# 毕业设计项目技术说明文档（图生文 / 文生图）

> 版本：基于当前代码仓库
> 核心目标：实现“图生文（反推提示词）+ 文生图（可控生成）”的双向闭环系统，并保证参数可解释、流程可追踪、结果可复现（条件下）。

---

## 1. 项目总体定位

本项目是一个 **FastAPI + React** 的图像智能系统，包含两个核心能力：

1. **图生文（/reverse）**
   上传图片，返回：
   - caption（自然语言描述）
   - structured（6块结构化提示词）
   - prompt（编译后的可生成提示词）
   - meta（模型、置信度、耗时）

2. **文生图（/generate）**
   上传 JSON（可包含 structured），结合可选参数（mode/seed/steps/cfg/style 等）生成图片，返回：
   - image_url
   - 生效参数与可解释元信息（effective_params、reproducibility、prompt_source）

核心入口：`backend/main.py:259`（/reverse）与 `backend/main.py:319`（/generate）。

### 3.1 reverse 结构（核心亮点）
`StructuredPrompt` 定义了 6 个块（`backend/schemas.py:66-74`）：

2. scene

## 6. 端到端流程总结（答辩可直接讲）

1. 用户上传图片 -> 前端校验 -> `/reverse`
2. 后端分析图片 -> 结构化归一 -> 编译 prompt -> 返回 JSON
3. 用户将 JSON 用于 `/generate`，并可覆盖参数
4. 后端按“表单 > structured.params > 默认值”归一化参数
5. 进入 `sd/gemini/proxy` 生成
6. 返回图片地址 + 生效参数 + 可复现等级

## 10. 一段可用于复试的总结话术

本项目的重点不在“调用了模型”，而在于建立了
**结构化数据 -> 参数归一化 -> 多模式生成 -> 元信息可解释** 的完整工程链路。
这条链路让系统具备三点能力：**可控、可复现（条件下）、可追踪**，并且通过测试验证了关键逻辑（兼容性、参数优先级、边界校验）。

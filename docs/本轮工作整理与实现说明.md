## 4. 关键设计解释（答辩可直接讲）

1) 为什么是“结构化 -> 编译 -> 参数透传”
- 结构化数据本身不等于可控，关键是要落到模型调用参数。
- 现在链路已打通，且通过 `effective_params` 可追踪“实际生效值”。

2) 为什么定义 `reproducibility=strong|best_effort`
- `sd + 固定 seed` 在同环境可较强复现。
- 云端/代理模式受服务端不可控因素影响，只能 best-effort。

3) 为什么风格预设做“增量注入”
- 避免风格词覆盖主体语义。
- 保留主体字段优先，减少“风格强但主体漂移”。
